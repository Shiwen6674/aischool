概念網絡分析工作台

這一版把原本偏工程測試頁的概念網絡工具，改成更像視覺分析工作台。左邊專注在文本與參數，中間專注在圖形本身， 右邊專注在閱讀與解釋。節點大小不再只看詞頻，而是根據 Strength、Betweenness、Closeness 綜合加權。

Input
詞(詞性) + 空白分隔
Visual Direction
Gephi-like Focus Map

文本與分析設定

先輸入斷詞文本，再決定節點保留範圍、共現強度、停用詞與節點加權方式。

格式建議：每個詞為 詞(詞性)，詞與詞之間用空白分隔。
限制每個節點保留幾條最強邊，避免變成毛球。
保留詞性
預設忽略：DE / SHI / T / DI / P / Nep / Nh / Neqa / Cbb
用逗號分隔。適合排除泛用字、贅詞或教學口頭詞。
節點權重公式

概念圖視覺化

節點大小反映綜合權重，邊粗細與透明度反映關聯強度。搜尋、高亮與點擊都會同步改變閱讀焦點。

節點數 0 保留進圖中的概念
邊數 0 經過過濾後的連線
句子數 0 成功解析的句子單位
有效詞數 0 過濾後保留的詞彙
平均連線度 0.00 每節點平均連到幾條邊
尚未建構概念圖 貼上已斷詞文本後，按下「建構網絡」。建議先用 20 到 40 個節點，圖會比較像分析圖，而不是毛線球。

閱讀面板

右邊不只是數值，而是幫你快速看懂圖：誰是核心概念、誰是橋樑概念、目前選到的節點代表什麼。

怎麼看這張圖
Strength 高：通常是文本核心概念。
Betweenness 高：通常是跨概念群的橋樑。
Closeness 高：通常位在整體網絡的中心位置。
尚未選取節點。點一下圖中的節點，這裡會顯示它的詞頻、中央性與相鄰概念。
